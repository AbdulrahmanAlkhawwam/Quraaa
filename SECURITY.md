# Security Policy

Thank you for helping keep **Quraaa** secure.

We take the security of our users, contributors, and infrastructure seriously. If you discover a security vulnerability, we appreciate your responsible disclosure.

---

# Supported Versions

Only the latest stable release receives security updates.

| Version | Supported |
| ------- | :-------: |
| 1.x.x   | ✅ |
| < 1.0   | ❌ |

---

# Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub Issues or Discussions.**

Instead, report vulnerabilities privately by emailing:

**security@quraaa.com**

If the email address is not yet available, please open a private GitHub Security Advisory (if enabled) or contact the project maintainers directly.

Please include as much information as possible:

- Description of the vulnerability
- Steps to reproduce
- Affected components
- Potential impact
- Screenshots or proof of concept (if applicable)
- Suggested mitigation (optional)

---

# What to Expect

After receiving your report, we aim to:

| Stage | Target Time |
|--------|-------------|
| Initial acknowledgment | Within 72 hours |
| Initial assessment | Within 7 days |
| Status updates | At least every 14 days |
| Security fix | Depends on severity |

Every report is carefully reviewed. If the issue is confirmed, we will work on a fix as quickly as possible.

---

# Coordinated Disclosure

Please allow us reasonable time to investigate and release a fix before publicly disclosing the vulnerability.

We will coordinate with you regarding:

- Validation
- Severity assessment
- Patch development
- Public disclosure timeline
- Credit (if desired)

---

# Scope

This policy covers vulnerabilities affecting:

- Mobile applications
- Backend services
- Authentication
- Authorization
- APIs
- Payment integration
- User data protection
- File storage
- AI service integrations
- Infrastructure owned by Quraaa

---

# Out of Scope

The following are generally considered out of scope:

- Social engineering attacks
- Physical attacks
- Denial-of-service (DoS/DDoS) testing
- Spam
- Vulnerabilities in third-party services
- Issues requiring compromised user devices
- Reports without a reasonable security impact

---

# Security Best Practices

We are committed to following industry best practices, including:

- HTTPS/TLS encryption
- JWT authentication
- Secure password storage
- Principle of least privilege
- Input validation
- Output encoding
- Secure secret management
- Dependency vulnerability monitoring
- Regular security updates

---

# Responsible Disclosure

We ask researchers to:

- Act in good faith.
- Avoid accessing or modifying user data.
- Avoid disrupting the service.
- Keep vulnerabilities confidential until a fix is available.
- Provide sufficient details to reproduce the issue.

---

# Recognition

We sincerely appreciate responsible security researchers who help improve Quraaa.

With your permission, we may acknowledge your contribution in the project's security acknowledgments after the issue has been resolved.

Thank you for helping make Quraaa safer for everyone.
